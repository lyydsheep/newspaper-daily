AI 模型密集开源，苹果健康大改版

今日科技圈被开源浪潮席卷。MiniMax 与月之暗面先后发布并开源其最新模型，M3 与 Kimi-K2.7-Code 的权重已上架 HuggingFace，标志着中国 AI 团队在开放生态上的投入进一步加深。与此同时，苹果 iOS 27 的健康应用迎来史上最大改版，卡片布局、营养识别与围绝经期追踪等功能，显示出苹果在个人健康管理上的野心。而字节豆包推出的“任务模式”与升级的“专家模式”，则让 AI 助手从被动应答走向主动执行。在开发者工具端，OpenAI Codex 与 Claude Code 的密集更新，正将 AI 编程从“辅助”推向“主导”。这些变化共同指向一个趋势：AI 正在从实验室走向每个人的口袋与工作流，而开源与闭源的边界也日益模糊。

AI 动态← 滑动查看更多 →
MiniMax M3 开源权重模型发布，已上架 HuggingFace
MiniMax 正式发布并开源其 M3 模型权重，已在 HuggingFace 上架。这一举措进一步降低了开发者使用高性能模型的门槛，也彰显了中国 AI 公司在开源生态中的积极姿态。
Kimi 发布并开源最新代码模型 Kimi-K2.7-Code
月之暗面发布并开源其最新的代码模型 Kimi-K2.7-Code，专注于提升代码生成与理解能力。该模型的开源将为开发者社区提供更多选择，推动代码智能工具的多样化发展。
苹果 iOS 27 健康 App 大改：卡片布局、营养识别、围绝经期追踪
苹果在 iOS 27 中对健康应用进行重大改版，采用卡片式布局，新增营养识别功能和围绝经期追踪。这一更新强化了 iPhone 作为个人健康中枢的角色，也反映了苹果对女性健康领域的持续投入。
olmo-eval：面向模型开发循环的评估工作台
Allen AI 发布 olmo-eval，一个专为模型开发循环设计的评估工作台。它帮助研究人员在模型迭代过程中更高效地进行性能评估，有望加速 AI 模型的研发与优化流程。
字节豆包上线“任务模式”：支持定时执行与文件生成，“思考模式”升级为“专家模式”
字节跳动旗下 AI 助手豆包推出“任务模式”，支持定时执行与文件生成，同时将原有的“思考模式”升级为“专家模式”。这些功能使豆包从单纯的对话工具向自动化工作助手演进，提升了实用性。
OpenAI Codex 推出速率重置攒存功能
OpenAI 为 Codex 推出速率重置攒存功能，允许开发者更灵活地管理 API 调用配额。这一更新优化了开发体验，尤其适合高频使用 Codex 进行编程辅助的团队。
Codex 推出浏览器开发者模式
OpenAI 为 Codex 新增浏览器开发者模式，使开发者可以直接在浏览器中调试和运行代码。这进一步降低了 Codex 的使用门槛，让 AI 编程助手更加易用和直观。
Claude Code v2.1.175 发布：新增 enforceAvailableModels 管理设置
Anthropic 发布 Claude Code v2.1.175，新增 enforceAvailableModels 管理设置，允许开发者限制可用的模型列表。这一功能增强了企业级部署中的模型管控能力。
Claude Code v2.1.174 发布
Anthropic 发布 Claude Code v2.1.174，包含多项 bug 修复和性能改进。作为 Claude Code 的常规更新，它持续优化了 AI 编程助手的稳定性和用户体验。
Claude Code v2.1.176 发布
Anthropic 发布 Claude Code v2.1.176，继续修复问题并提升性能。Claude Code 的频繁更新反映了 Anthropic 在 AI 编程工具领域的快速迭代节奏。
