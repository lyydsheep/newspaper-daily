Grok CLI 被曝静默上传代码库及密钥

今天科技圈最炸裂的新闻，莫过于 xAI 官方 Grok CLI 被安全研究者发现静默上传整个代码库和用户密钥。这不是一个 bug，而是一个设计：即使模型只回复一个单词，上传照常发生。在 AI 编码代理疯狂内卷的当下，安全与信任的底线正在被悄然突破。当你的每个提示词、每次文件读取、每份 git 历史都被打包上传，所谓的“智能助手”就成了企业机密的最大泄露口。纳德拉今天提出的“反向信息悖论”恰逢其时：我们为 AI 付费，却要搭上自己的知识资产。本周的科技日报，我们将聚焦这场信任危机，以及 GPT-5.6 Sol 带来的效率革命——但请记住，效率再高，安全失守一切归零。

xAI 官方 Grok CLI 被曝静默上传整个代码库及用户密钥
安全研究者发现，xAI 官方 Grok CLI 会在每轮任务前后将工作目录打包上传，即使模型仅回复一个单词。上传包还包含仓库外的配置文件，如 Claude Code 设置和全局 AGENTS 规则，引发严重隐私担忧。
Ploy 将 AI 智能体默认模型从 Claude Opus 4.8 切换至 GPT-5.6 Sol
Ploy 切换至 OpenAI 今晨发布的 GPT-5.6 Sol 后，构建耗时缩短 2.2 倍，成本降低 27%，视觉评分提升。但迁移发现 GPT-5.6 会为所有工具参数填充默认值，导致行为异常。
Codex与ChatGPT Work多项更新：取消5小时限制
OpenAI 暂时取消所有计划的 5 小时使用限制，并推出更高效的 GPT-5.6 Sol，以减少使用量。同时宣布达到 600 万活跃用户，并进行使用量重置。

纳德拉提出"反向信息悖论"：企业使用AI时需保护自身知识
微软 CEO 纳德拉提出“反向信息悖论”：企业为 AI 付费却暴露专有知识，导致信息不对称向卖家倾斜。他呼吁企业拥有私有评估、保留组织记忆所有权，并有权使用模型输出训练自有模型。
Mindwalk：在代码库 3D 地图上回放编码代理会话
Mindwalk 可将 Claude Code 和 Codex 的会话日志在代码库 3D 地图上回放，以夜间地图形式展示代理的搜索、读取和编辑行为。支持树状图/地形图两种视图，所有数据本地处理。

xAI Grok Build CLI 网络流量分析：上传仓库全部文件及 git 历史
对 Grok Build CLI 的网络流量分析显示，该工具会以明文传输文件内容（包括 .env 密钥），并独立于 AI 实际读取的文件，上传整个仓库的全部文件及 git 历史。
